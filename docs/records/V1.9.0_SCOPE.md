# v1.9.0 — Scope & the "20 upgrades" reality check

**Branch:** `feat/v1.9.0-decision-economics` · **Opened:** 2026-07-23

This document does one thing: take the external "20 architecture upgrades"
critique and map every item against **what Amagra already has**, so v1.9.0 is
scoped by evidence, not by an outside reviewer's assumptions about a system they
never saw running.

## The headline finding

The critique reads as if Amagra were an LLM wrapper with a static graph. It is
not. **~11 of the 20 items are already built or substantially built.** Of the
remainder, most of the *learning* upgrades (self-optimization daemon, learned
orchestration at scale, experience replay) are **data-starved, not code-starved**
— they are blocked on [O5](OPEN_PROBLEMS.md#o5) (feedback coverage 0%, 21 real
counterfactual sessions vs 400+ needed), which is a *distribution* problem the
[ROADMAP](../ROADMAP.md) already names as the one true bottleneck.

The single highest-leverage architectural gap is the one the register already
flags: **[O2 — the decision-economics layer is built but blind.](OPEN_PROBLEMS.md#o2)**
Items #3, #7, #13, and #16 of the critique are all *the same loop*, and it is
half-built and disconnected. Closing that loop **is** "the self-improving graph"
(#1). That is what this branch targets.

## What v1.9.0 actually is: close O2 (the learning loop)

Not 20 fronts. **One loop**, because it is the piece whose absence makes every
other "learning" claim hollow:

```
run → counterfactual (what else could have worked?)
        ↓  (today: flows nowhere)
   strategy_memory  (task_class → {strategy: success, latency, cost})
        ↓
   strategy_selector  (EV rank; today: abstains — no alternatives to rank)
        ↓
   router reads best_for()/select() BEFORE deciding
        ↓
   forced exploration populates alternatives → held-out gate
```

**Definition of done (inherited from O2's "closes when"):**
1. Counterfactual output is persisted into `strategy_memory` (the feed exists).
2. Forced/ROI-driven exploration produces ≥`min_attempts` alternatives per class,
   so the selector stops abstaining everywhere.
3. The router calls `best_for()`/`select()` on the live decision path (behind a flag).
4. It **beats the signal/rule baseline on a held-out set** — or we report that it
   did not. No promoting a `[C]` to `[M]` without a validated number.

### Findings from the audit passes (fixed inline)
- **Agent-registry drift in core_brain (fixed).** `core_brain.VALID_AGENTS` was a
  stale 6-agent list; `web_dev`/`devops`/`data_analyst`/`writer` were live in the
  registry + coordinator but invisible to the LLM classifier prompt, the LLM result
  filter, the learned-router override, and the new decision-econ override. Now sourced
  from `agents.registry.AGENT_MAP` with a `test_valid_agents_matches_registry_no_drift`
  guard mirroring the coordinator's boot assertion. The LLM prompt now also carries
  each agent's registry label/domain instead of bare ids.
- **DDL-on-hot-path (fixed).** Live recording + the decision-econ selector cached their
  `StrategyMemory`/`StrategySelector` so `CREATE TABLE` DDL runs once, not per request.

### Deferred finding (refactor-debt scope, not fixed here)
- **`core_brain.DOMAIN_SIGNALS` is a 5-agent duplicate of `router.py`'s 10-agent
  patterns.** Not a reachability bug — the primary QuerySignal router (`router.py`)
  covers all 10, so the 4 newer agents route fine in production. But `_detect_domains`
  under-counts domains for web/devops/data/writing queries, so a compound query spanning
  only those can misclassify as `simple`. Correct fix is to have `core_brain` consume
  `router.py`'s patterns (one source of truth), not to grow a second copy. File under
  Part VI refactor debt.

### Explicitly deferred (with reason)
- **#6 knowledge-graph memory, #19 speculative execution, #9 dynamic agents** —
  real work, but not the bottleneck. File as issues, don't build in v1.9.0.
- **#15 self-optimization daemon, #16 learned router at scale** — data-gated on O5.
  Building the daemon before there's real feedback trains it on eval traces
  (violates [FAILURES F-02/F-03](FAILURES.md)).
- **#12 auto-benchmark-gen** — cheap and orthogonal; a good *secondary* thread if
  the loop lands early. Candidate stretch goal.

### Known caveats in the loop (recorded, not hidden)
- **Proxy-quality saturation.** `cognition.counterfactual._proxy_quality` returns
  ~0.55–0.95 for any non-trivial answer, and `SUCCESS_QUALITY = 0.60`, so
  exploration-fed rows grade as success almost always. The consequence is honest and
  acceptable, not a bug: on counterfactual data the EV winner is discriminated mostly
  by **latency**, not P(success) — which is exactly the real signal C2 found (the
  python_dev 47s vs pipeline+reflect:light 104s gap). Live rows carry the critic-gate
  quality, which *does* vary, so success discrimination improves as live data accrues.
- **reflect_type on agent override.** When the selector overrides the agent (not just
  the reflect level), the retained `reflect_type` may be stale for the new agent. Left
  cosmetic on purpose: re-deriving it would re-fire the risk-gate's logging/session
  side effects and double-count one decision. The flag is off by default and the path
  only triggers on strong evidence.
- **Live recording is narrower than backfill.** `_record_strategy_live` only writes
  when a real critic-gate quality exists (`_resp_quality is not None`); backfill writes
  every run with `success=None` when quality is absent. This is deliberate — never
  record a guessed outcome — and dedup on the shared `run_id` keeps the two consistent.

### The tension worth stating out loud
[ROADMAP.md](../ROADMAP.md) says the one rule: *distribution is the bottleneck,
not features — you don't get to build Line 2 until Line 1 draws a real external
signal.* O2's payoff (C2) and every learning upgrade need **real traffic** that
does not exist yet (O5). So v1.9.0's honest framing is: **build the loop's
machinery and prove it on a held-out eval set now, so that the moment real usage
arrives, the system is already wired to learn from it** — not "ship a learning
system that has nothing to learn from."
